# 新闻数据源说明

## ⚠️ 重要说明

**本项目已移除所有备用数据，仅使用真实抓取的新闻数据。**

由于浏览器CORS（跨域资源共享）限制，在本地开发环境可能无法直接抓取新闻。如果显示"无法获取新闻"，请使用以下方法之一：

## 📰 真实新闻来源

系统会实时从以下源抓取新闻：

### 热榜类
- **百度热搜** - top.baidu.com (实时热搜)
- **百度财经** - top.baidu.com (财经热搜)
- **知乎热榜** - zhihu.com (热门问答)
- **微博热搜** - weibo.com (实时热点)

### 科技财经类
- **36氪** - 36kr.com (科技创业)
- **虎嗅** - huxiu.com (商业科技)
- **雪球** - xueqiu.com (投资社区)
- **东方财富** - eastmoney.com (财经门户)

## 🚀 解决方案

### 方案一：部署到服务器（推荐）

```bash
# 部署到 Vercel（免费）
npm install -g vercel
vercel

# 服务器端没有CORS限制，可以成功抓取
```

### 方案二：配置新闻API

创建 `.env.local` 文件：

```env
# GNews API（免费每天100条）
# 获取地址：https://gnews.io/register
GNEWS_API_KEY=your_key_here

# NewsAPI（免费每天100条）
# 获取地址：https://newsapi.org/register
NEWS_API_KEY=your_key_here

# Currents API（免费每天200条）
# 获取地址：https://currentsapi.services/register
CURRENTS_API_KEY=your_key_here
```

### 方案三：使用代理

在项目中配置代理服务器绕过CORS限制。

## 🎯 为什么本地无法抓取？

**CORS限制**：浏览器安全策略禁止网页直接请求其他域名的资源。

**解决方案对比：**

| 环境 | CORS限制 | 能否抓取 |
|------|---------|---------|
| 本地开发（浏览器） | ✅ 有限制 | ❌ 可能失败 |
| 服务器（Node.js） | ❌ 无限制 | ✅ 可以成功 |
| Vercel/Railway | ❌ 无限制 | ✅ 可以成功 |
